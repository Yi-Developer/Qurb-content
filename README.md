# qurb-content

Скачиваемый контент для приложения **Qurb**: переводы Корана, названия стран и городов.
Файлы раздаются бесплатно через **jsDelivr** (CDN поверх GitHub). Приложение скачивает
нужный файл один раз, распаковывает и кэширует на устройстве — дальше работает оффлайн.

Всё, что здесь лежит, — это **тяжёлые опциональные данные** (42 перевода Корана и т.п.).
Мелкие всегда-нужные данные (дуа, азкары, праздники) вшиты прямо в приложение и сюда
не входят — так и должно быть.

---

## Структура

```
qurb-content/
├── manifest.json            ← корневой манифест (список категорий)
├── quran/
│   ├── editions.json        ← список ВСЕХ изданий Корана с метаданными (главный файл)
│   ├── arabic.json.gz        ← арабский текст (Усмани, Хафс)
│   ├── translit.json.gz      ← транскрипция латиницей
│   └── translations/         ← 42 перевода, по файлу на издание
│       ├── t_russian_kuliev.json.gz
│       ├── t_english_saheeh.json.gz
│       └── ...
└── geo/
    ├── index.json           ← манифест гео-данных
    ├── countries.json.gz     ← 244 страны на 5 языках
    └── cities/               ← города по языкам (ar, az, ja, kk, ky, ru, tg, tr, uz)
        ├── ru.json.gz
        └── ...
```

## Форматы файлов

| Файл | Формат внутри (после gunzip) |
|------|------|
| `quran/arabic.json.gz`, `translit.json.gz`, любой перевод | `{ "сура:аят": "текст", ... }` — 6236 ключей, напр. `"2:255"` |
| `geo/countries.json.gz` | `{ "EnglishName": { "ru":.., "ar":.., "kk":.., "tr":.., "ja":.. }, ... }` |
| `geo/cities/<lang>.json.gz` | `{ "EnglishName": "ЛокализованноеИмя", ... }` |

Все файлы — **gzip + JSON (UTF-8)**. Держим их сжатыми: меньше трафик, jsDelivr отдаёт
как есть, приложение распаковывает у себя.

## Ссылки jsDelivr

После публикации на GitHub (репозиторий `USER/qurb-content`) любой файл доступен по адресу:

```
https://cdn.jsdelivr.net/gh/USER/qurb-content@1.0.0/<путь>
```

Примеры:
```
.../gh/USER/qurb-content@1.0.0/quran/editions.json
.../gh/USER/qurb-content@1.0.0/quran/translations/t_russian_kuliev.json.gz
.../gh/USER/qurb-content@1.0.0/geo/cities/ru.json.gz
```

Используй **тег версии** (`@1.0.0`), а не `@latest` или `@main` — так jsDelivr кэширует
навсегда и файлы грузятся быстрее. Выпустил новую версию контента → поднял тег.

## Как работает механизм «скачать перевод»

1. Приложение один раз читает маленький `quran/editions.json` (пара десятков КБ).
2. Строит по нему меню: группирует издания по языку (`language_native`), показывает переводчика.
3. Пользователь жмёт «Скачать» → приложение качает один `.json.gz` из `file`.
4. Распаковывает (gunzip), сохраняет на устройство, помечает как скачанный.
5. Дальше читает с устройства — CDN больше не дёргается.

`editions.json` содержит для каждого издания: `id`, `file`, `lang`, `language_en`,
`language_native`, `translator`, `script`, `ayahs`, `size_gz` — этого хватает, чтобы
показать список и прогресс скачивания, ничего не докачивая заранее.

## Обновление контента

- Добавить перевод: положи `t_<язык>_<издание>.json.gz` в `quran/translations/`,
  допиши запись в `editions.json`, подними версию тега. Обновление приложения в сторе
  **не требуется** — оно подхватит новый пункт из манифеста.

## ⚠️ Права на переводы

Часть переводов Корана защищена авторским правом переводчика. Перед публикацией убедись,
что у тебя есть право распространять именно эти издания (многие переводы Rowwad / Montada /
King Fahd Complex распространяются свободно для да'ва, но условия стоит проверить по каждому).
Арабский текст, транскрипция и названия стран/городов ограничений не имеют.
